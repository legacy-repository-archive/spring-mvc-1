# 📕 서블릿   
## 📖 서블릿 탄생 배경    
클라이언트가 서버에게 요청을 보낼때는 `HTTP Message`를 보낸다.           
     
이때, 서버에서는 아래와 같은 작업을 해야한다.   

* 서버 TCP/IP 연결 대기, 소켓 연결   
* HTTP 요청 메시지를 파싱해서 읽기   
* POST 방식, /save URL 인지  
* Content-Type 확인 
* HTTP 메시지 바디 내용 파싱   
    * username, age 데이터를 사용할 수 있게 파싱  
* 저장 프로세스 실행 
* 비즈니스 로직 실행
    * 데이터베이스에 저장 요청 
* HTTP 응답 메시지 생성 시작 
    * HTTP 시작 랑니 생성
    * Header 생성
    * 메시지 바디에 HTML 생성에서 입력 
* TCP/IP에 응답 전달, 소켓 종료
